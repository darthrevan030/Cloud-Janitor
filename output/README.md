# output/

Runtime artifacts generated by the Cloud Janitor pipeline. Most files here are gitignored — they reflect transient infrastructure state.

## Directory Structure

```
output/
├── findings_store.json       # Scan results (FinOps + SecOps findings)
├── remediation.tf            # Combined remediation HCL for current run
├── logs/
│   ├── audit.log             # Append-only audit trail (JSONL)
│   ├── agent_reasoning.log   # Agent decision trace (JSONL, truncated each run)
│   └── scheduler.log         # Cron scheduler log (rotating, 10MB max)
├── policies/                 # Incident-response policy JSON files
│   └── policy-*.json
├── rollbacks/                # Per-resource rollback HCL
│   └── <resource_id>.tf
└── README.md
```

## `findings_store.json`

- **Written by**: FinOpsAuditor (creates fresh) and SecOpsGuard (appends)
- **Read by**: RemediationArchitect, SavingsTracker, Streamlit UI
- **Schema**: See `agents/README.md` for full field documentation
- **Gitignored**: Yes — regenerated on every scan

## `remediation.tf`

- **Generated by**: RemediationArchitect agent
- **Overwritten** on each scan cycle — never edit manually
- **Applied via**: `tflocal apply -auto-approve` after explicit approval through the Approval Gate
- **Gitignored**: Yes

### Lifecycle

1. RemediationArchitect reads `findings_store.json` and runs dependency checks
2. For each actionable finding, generates remediation HCL
3. All remediations written to `remediation.tf`, replacing any previous content
4. Pre-remediation hook validates the file (`tflocal validate`)
5. Approval Gate displays HCL and waits for `APPROVE <resource-id>`
6. On approval, submitted to `tflocal apply`

## `logs/`

| File | Format | Lifecycle | Size Limit |
|------|--------|-----------|------------|
| `audit.log` | JSONL (append-only) | Never truncated | Unbounded |
| `agent_reasoning.log` | JSONL | Truncated at start of each audit run | Unbounded within a run |
| `scheduler.log` | Plain text | RotatingFileHandler | 10MB × 3 backups |

## `policies/`

JSON files generated by the IncidentPolicyGenerator agent. Each file contains a security policy recommendation for a specific finding (e.g. open port, missing encryption).

Named as `policy-<description>.json`. Gitignored.

## `rollbacks/`

Per-resource rollback Terraform files. See `rollbacks/README.md` for details.

## What's Gitignored

All runtime artifacts are excluded from version control:

- `findings_store.json`, `scan_history.json`, `savings_ledger.json`
- `logs/*.log` (all log files)
- `rollbacks/*` (except `.gitkeep`)
- `policies/*.json`
- `remediation.tf`
